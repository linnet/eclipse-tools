Quickmarks Plugin: Switching Styles

"M" company style

If you prefer to work with Quickmarks "M company Style" (only 3 key bindings instead of 20, but 'toggling' through your scope only, no 'mark move') you can configure this plugin to do that. A mild form would be to have the Quickmarks 1-9 to still work "B company style", and use Quickmark 0 to work "M company Style" (the best of both worlds):

  • choose the option to have dups all the time
  • redefine the binding for Set/Delete Mark 0 to, say, Shift+Ctrl+F12, the binding for Go to next Mark 0 to Ctrl+F12 and the binding for Go to previous Mark 0 to Shift+F12
  • set the scope to document (or whatever you prefer)

Now you can use the Quickmark 0 in your preferred style, and optionally still use all other nine Quickmarks.

An extreme form would be to even unbind all the keys for Quickmarks 1-9 additionally. As this is much work to do we deliver another plugin.xml.msstyle. If you really want to only work with this (e.g. to get the key bindings available for other actions), then:

  • if you run Quickmarks under Eclipse 3 then be sure to have read this very important note and follow the procedure outlined, to avoid any pitfalls!
  • stop Eclipse
  • rename plugin.xml to plugin.xml.original
  • copy plugin.xml.msstyle over plugin.xml
  • start Eclipse
  • open the Quickmark preferences and check the option to have dups all the time
  • set your scope preference to document (for really pure MS style) or the scope you prefer
  • better set the delete option to 'same line'
  • edit the Quickmarks name to, say, "Quickmark at line {1}"
  • be sure to (again) configure all perspectives to enable Quickmarks, where you want to have them

When you have done this, the menu has changed:

Compare to the default. The key binding section in the preferences is changed accordingly: there are only 3 actions left to bind keys to. Other changes are:

  • Now there are also toolbar buttons.
  • The Quickmark icon left to any editor shows as an arrow pointing right.
  • The plugin doesn't requiere early startup anymore and isn't listed there.

Pure "B" company style

If you prefer to work with the original key bindings Ctrl+0-9 for jumping to a Quickmark instead of the default ones Alt+0-9, then you might manually redefine all the bindings yourself.

Or you may use the delivered *.bstyle files:

  • if you run Quickmarks under Eclipse 3 then be sure to have read this very important note and follow the procedure outlined, to avoid any pitfalls!
  • stop Eclipse
  • rename plugin.xml to plugin.xml.original
  • copy plugin.xml.bstyle over plugin.xml
  • rename plugin.properties to plugin.properties.original
  • copy plugin.properties.bstyle over plugin.properties
  • start Eclipse

Note however, that there are 2 key binding conflicts:

  1. 'Go to Quickmark 1' bound to Ctrl+1 is hidden in the Java editor by 'Quickfix' bound to the same key
  2. 'Go to Quickmark 2' bound to Ctrl+2 is hidden in 'Emacs' configuration by 'Set mark' bound to the same key

Especially the 'Quickfix' is so important when developing Java code, that most people might not consider rebinding that action. If you are one of them you must live without Quickmark 1 or (inconsistently) jump to it with Alt+1 bound to 'Go to previous Quickmark 1' or by means of the menu. If you decide to live without Quickmark 1 then you better unbind 'Set Quickmark 1' to not accidentally set it anywhere.

Suggestion for rebinding "Quickfix"

I always wondered, why Ctrl+1 was used for "Quickfix", a much better binding seems Ctrl+Q to me, which is even a little bit better reachable with short fingers. But Ctrl+Q is bound to "Go to Last Edit Location" ... now what, where is the 'Q' in that actions name??! And it is the only action using any modifier combination together with the letter 'Q'. And it seems to be closely related to "Back" and "Forward", which are bound to Alt+Arrow_Left and Alt+Arrow_Right respectively. And Alt+Arrow_Up is free! So I suggest:

  • rebind "Go to Last Edit Location" to Alt+Arrow_Up
  • rebind "Quickfix" to the now available Ctrl+Q

Suggestion for rebinding "Set mark" in "Emacs" configuration

If you are an Emacs purist, then you probably havn't this extension installed anyway, have you? If you work mostly with the 'point and mark' concept of Emacs and only occasionally use other Quickmarks then you might again just don't use Quickmark 2. Else:

  • rebind "Set mark" to Ctrl+M, which is unused.
